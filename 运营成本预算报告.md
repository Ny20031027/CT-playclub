# CT电竞（PlayClub）微信云托管运营成本预算报告

> 本报告基于微信云托管官方定价，结合项目实际业务场景进行真实模拟估算。
> 编制日期：2026年7月19日

---

## 一、项目概述

CT电竞（PlayClub）是一个游戏代练/陪玩服务平台，基于 Django + MySQL + 微信小程序 架构，核心功能包括：

| 模块 | 说明 |
|------|------|
| 用户管理 | 客户、打手（员工）双角色体系 |
| 订单系统 | 发布→接单→进行中→完成→评价，全流程管理 |
| 排班系统 | 打手排班与时段管理 |
| 财务系统 | 收入统计、提现管理 |
| 数据统计 | 运营数据看板 |
| 文件上传 | 头像、订单图片等 |

## 三、资源配置方案

### 3.1 推荐配置

| 资源 | 配置 | 说明 |
|------|------|------|
| **容器（服务）** | 1 核 2GB | Django + Gunicorn，3 个 worker |
| 实例副本数 | 最小 0，最大 3 | 自动扩缩容，无流量时零成本 |
| **MySQL** | 自动弹性 0.25-16 CCU | Serverless，按需伸缩 |
| 数据库存储 | 5GB（初期） | 按实际存储量计费 |
| 对象存储 | 5GB | 用户头像、订单图片 |
| 静态资源 | 20MB | 前端页面、JS、CSS |

### 3.2 为什么选择此配置

- **1 核 2GB**：Django REST Framework 单实例可处理 50+ 并发请求，完全满足 1000 DAU
- **自动缩容到 0**：凌晨无流量时完全不计费，这是云托管相比传统服务器的核心优势
- **MySQL 弹性伸缩**：Serverless 架构，闲时自动降为 0.25 CCU，忙时自动升至所需算力

## 七、与传统服务器方案对比

以 1000 DAU 为基准，对比传统云服务器方案：

| 方案 | 月费用 | 优势 | 劣势 |
|------|--------|------|------|
| **微信云托管（推荐）** | **~240 元** | 自动扩缩容、免运维、高可用、免费额度 | 按量付费，流量大时成本上升 |
| 腾讯云 ECS 2核4G + 云数据库 | ~350 元 | 固定成本可预测 | 需自行运维、无法自动扩缩容 |
| 腾讯云轻量 2核2G + 自建MySQL | ~150 元 | 成本最低 | 需自行运维、单点故障风险 |
| 阿里云 ECS 2核4G + RDS | ~400 元 | 生态丰富 | 成本最高 |

**结论**：微信云托管在 1000 DAU 规模下，成本低于传统 ECS + 数据库方案约 30%，且具备自动扩缩容和免运维优势。

## 九、投资建议

### 9.1 核心结论

1. **极低的启动成本**：首 3 个月仅需约 51 元/月，年运营成本约 2,300 元
2. **弹性可扩展**：从 100 到 10,000 DAU，无需迁移架构，自动扩缩容
3. **免运维**：无需服务器运维人员，团队专注业务开发
4. **微信生态深度集成**：天然支持小程序登录、支付、消息推送

### 9.2 风险提示

1. **流量突增风险**：若出现爆款传播，公网流量费用可能短期上升（可通过限制外网访问缓解）
2. **数据库存储增长**：长期运营后存储费用会持续增长，建议定期清理过期数据
3. **免费额度到期**：第 4 个月起费用会从 ~51 元跳至 ~240 元，需提前规划

### 9.3 建议配置

| 阶段 | DAU | 推荐配置 | 月预算 |
|------|-----|---------|--------|
| MVP 验证期 | < 500 | 0.5核1G + MySQL 0.25CCU | 100 元 |
| 初创运营期 | 500-2000 | 1核2G + MySQL 0.5CCU | 300 元 |
| 规模增长期 | 2000-5000 | 2核4G + MySQL 1CCU | 800 元 |
| 成熟稳定期 | 5000-10000 | 4核8G + MySQL 2CCU | 2,000 元 |
